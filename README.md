# Проект интерактивной 3D-визуализации куба

## Обзор проекта
Данный проект представляет собой интерактивное графическое приложение, разработанное на языке **C++** с использованием библиотек **SFML** и **OpenGL**. Проект является технической демонстрацией рендеринга в реальном времени, геометрических преобразований и обработки пользовательского ввода. Основные возможности включают управление 3D-кубом с пронумерованными гранями, динамическое освещение, циклическое изменение цвета и сложную анимацию «развертки», превращающую куб в плоскую фигуру.

---

## Технологический стек
* **SFML (Simple and Fast Multimedia Library):** отвечает за создание окна, управление контекстом OpenGL и обработку ввода с клавиатуры в реальном времени.
* **OpenGL:** используется для низкоуровневого рендеринга, работы с буфером глубины и расчета освещения.
* **GLU (OpenGL Utility Library):** применяется для настройки перспективной проекции и позиционирования камеры.

---

## Основные компоненты

### 1. Структуры данных
* **Структура Rotation:** инкапсулирует состояние поворота куба по осям X, Y и Z, используя углы Эйлера с плавающей точкой.
* **Массивы вершин и граней:** геометрия куба определена через массив из 8 вершин и массив индексов для 6 граней, что обеспечивает эффективную отрисовку полигонов.

### 2. Графический движок
* **drawCube():** основная функция отрисовки. Она итерирует по граням куба, применяя к каждой индивидуальные трансформации. Функция учитывает нормали вершин для расчета освещения и динамически вычисляет цвета граней, используя синусоидальные функции, привязанные к текущим углам поворота.
* **drawFaceNumber():** отрисовывает каркасные цифры (1–6) на каждой грани. Для предотвращения **Z-конфликта** (мерцания при наложении полигонов) цифры выводятся с небольшим смещением координат относительно поверхности грани.
* **setupOpenGL():** настраивает трехмерную среду, включая:
    * Перспективную проекцию (угол обзора $45^\circ$).
    * Тест глубины для корректного отображения перекрывающихся объектов.
    * Источник света (`GL_LIGHT0`) с фоновой (ambient) и диффузной составляющими.

### 3. Геометрические трансформации
* **Управление стеком матриц:** использование `glPushMatrix()` и `glPopMatrix()` позволяет применять локальные вращения и перемещения. Это критически важно для реализации функции развертки.
* **Анимация развертки:** при активации грани куба «откидываются» относительно базовой грани (Face 5). Это достигается путем переноса оси вращения на ребро грани, поворота на угол развертки и последующего возврата в исходную систему координат.
* **Система камеры:** реализована функция масштабирования (зум) путем изменения координат точки наблюдения в функции `gluLookAt`.

## Логика управления и физика

### Обработка ввода
Приложение разделяет ввод на **потоковый** (опрос состояния клавиш каждый кадр для плавного движения) и **событийный** (реакция на одиночное нажатие).

| Категория | Клавиша(и) | Действие |
| :--- | :--- | :--- |
| **Движение** | WASD / Стрелки | Вращение по осям X и Y |
| | Q / E | Вращение по оси Z |
| **Модификаторы скорости** | Left/Right Shift | Ускорение вращения (в 3 раза) |
| | Left/Right Ctrl | Замедление вращения (в 0.3 раза) |
| **Переключатели логики** | C | Анимация развертки (сложить/разложить) |
| | Space | Режим отображения (заливка / каркас) |
| | T | Автоматическое вращение |
| **Управление видом** | R / L | Приблизить / Отдалить (зум) |
| | Backspace | Сброс всех углов поворота в 0 |
| **Калибровка** | +/- (Equal/Hyphen) | Изменение базовой скорости вращения |

### Независимость от частоты кадров
Для вычисления времени кадра (`deltaTime`) используется `sf::Clock`. Это гарантирует, что анимация развертки и автоматическое вращение происходят с одинаковой скоростью вне зависимости от производительности аппаратного обеспечения.

## Жизненный цикл выполнения
1. **Инициализация:** создание окна SFML и настройка контекста OpenGL.
2. **Опрос ввода:** проверка системных событий и текущего состояния клавиш.
3. **Обновление состояния:**
    * Интерполяция переменной `currentUnfoldAngle` в сторону $0^\circ$ или $90^\circ$.
    * Обновление углов поворота на основе активных клавиш и коэффициентов скорости.
4. **Отрисовка:**
    * Очистка буферов глубины и цвета.
    * Установка матрицы камеры с учетом `zoomFactor`.
    * Применение глобального вращения.
    * Вызов `drawCube()`, который обрабатывает шарнирное движение граней и рендерит геометрию.
5. **Вывод изображения:** смена заднего и переднего буферов (Double Buffering) для отображения кадра в окне.
